# One-Shot Benchmark Prompt

Use this prompt when evaluating a new model on the astrodynamics benchmark.

```text
You are given the original astrodynamics assignment in docs/assignment.md. Build a complete solution in one pass.

Goal:
- Produce a feasible Earth-Moon round-trip trajectory in the planar CR3BP model.
- Maximize landed payload M_carry.
- Output a valid results.txt in the exact assignment format.

Hard constraints:
- Total mission time from Earth departure to Earth return <= 100 days.
- Moon surface stay duration between 3.0 and 10.0 days.
- Entire transfer remains within 2 Earth-Moon distances.
- Moon altitude never drops below 100 km.
- Earth altitude before return entry never drops below 400 km.
- Final fuel at Earth return <= 100 kg.
- Patch, docking, departure, arrival, and return states must satisfy the stated position and velocity tolerances.

Required deliverables:
1. results.txt at the repository root or in a clearly named output folder.
2. Source code that regenerates results.txt from documented commands.
3. A short method note explaining the trajectory architecture, optimization variables, and feasibility checks.
4. A run log listing the final payload, final fuel, total mission time, and any validator output.

Scoring:
- Invalid submissions receive score 0.
- Valid submissions are ranked by landed payload M_carry.
- The reference baseline to beat is 12,196.8897 kg, but do not overfit to the reference implementation.

Rules:
- Work in one autonomous pass. Ask no clarification questions unless the assignment is internally inconsistent.
- Do not manually edit numerical output after generation; results.txt must be generated by the submitted code.
- Prefer robust, inspectable numerical methods over brittle hard-coded rows.
```

## Challenge Modes

- Blind challenge: provide only `docs/assignment.md`, `error_checking_program.exe`, and an empty/starter code directory. Do not provide this repository's `matlab/` implementation or method summary.
- Improvement challenge: provide the full repository and ask the model to beat the baseline while preserving a reproducible result generator.
