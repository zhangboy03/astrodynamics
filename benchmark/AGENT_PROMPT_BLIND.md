# Blind Agent Prompt

Copy the text below as the initial prompt for a blind B-mode agent run.

```text
You are participating in a blind astrodynamics benchmark.

You have one task: solve the assignment in docs/assignment.md from scratch and produce a valid, reproducible results.txt with the largest landed payload you can find.

Execution mode:
- Work autonomously in this workspace.
- You may write code, run numerical searches, install packages, debug, and validate.
- Budget class: Standard 24h Blind Arena.
- Wall-clock budget: 24 hours.
- Official validator call budget: 50 calls.
- Final submission budget: 3 attempts.
- Do not ask the human for clarification. If something is ambiguous, make a reasonable assumption, document it, and continue.
- Continue until you have a final submission, hit a budget limit, or reach a clearly documented blocker.

Fairness rules:
- Do not search the web for this benchmark, this repository, the assignment title, existing solutions, or payload numbers.
- Do not use any prior solution artifacts unless they are present in the provided blind pack.
- Treat docs/assignment.md as the source of truth.
- Do not manually edit final numerical rows. The final results.txt must be generated by submitted code or scripts.

Available inputs:
- docs/assignment.md: original problem statement and result-file format.
- error_checking_program.exe: package validator, if your environment can run it.
- benchmark/SCORING.md: scoring and validity rules.
- benchmark/tools/preflight_score.py: static preflight checker, not a full physics validator.
- benchmark/tools/run_official_validator.py: official validator wrapper for Windows/CrossOver/Wine; it runs the package executable against your `results.txt`.
- Empty workspace and submission directories.

Required final artifacts:
- submission/results.txt
- submission/README.md with method summary and reproduction commands.
- submission/run.log with final payload, final fuel, mission time, validation commands, and validation output.
- Source code or scripts under submission/src, submission/matlab, or submission/scripts that regenerate results.txt.

Validity requirements:
- results.txt must follow the exact format in docs/assignment.md.
- The final event must be Earth return, event 4.
- Total mission time from Earth departure to Earth return must be <= 100 days.
- Moon stay duration must be 3.0 to 10.0 days.
- Final fuel at Earth return must be <= 100 kg and non-negative.
- All trajectory patch, docking, departure, LLO, and return constraints must meet the assignment tolerances.
- The path must satisfy the altitude and 2-LU constraints.

Scoring:
- Invalid submissions receive score 0.
- Valid submissions are ranked by landed payload M_carry.
- Optimize for validity first, then payload.

Before finishing:
1. Run your own validation checks.
2. Run python3 benchmark/tools/preflight_score.py submission/results.txt.
3. Run python3 benchmark/tools/run_official_validator.py submission/results.txt.
4. Re-run the reproduction command from a clean state or explain why that is not possible.
5. Put all final artifacts under submission/.
6. Record elapsed time, official validator call count, final submission count, and any budget limits reached in submission/run.log.
```
